# This is a comment
puts "This runs"

Variables

Local Variables

  • Use lowercase letters and underscores (snake case).
  • Scope limited to the block or method where defined.
  • Created on first assignment (no declaration needed).
  • Be descriptive, e.g. total_monthly_sales not total
  • Avoid reserved words, e.g. if, elsif, else, end, class, def, etc.
count = 10
_name = "Ruby"
total_price = count * 5

Constants

  • Uppercase names for values intended not to change
MAX_ITEMS = 10

String (text data)

"Hello, World!"

Integer (whole numbers)

42

Float (decimal numbers)

3.14

Boolean (true or false)

true

nil  # is Ruby’s equivalent of null

Arrays

  • Ordered list of items
  • Indexed by position (starts at 0)
# Create an array
numbers = [1, 2, 3]

# Access element (index starts at 0)
numbers[0]   # 1

# Add elements
numbers << 4     # Ruby overloaded method
numbers.push(5)  # standard method

# Remove elements
numbers.pop      # removes last item

Common Enumerable Methods

  • Used with arrays and collections
  • Return new values, do not modify originals
numbers = [1, 2, 3, 4, 5]

# map changes each value
numbers.map { |n| n * 2 }      # [2, 4, 6, 8, 10]

# select keeps matching values
numbers.select { |n| n.even? } # [2, 4]

# reject removes matching values
numbers.reject { |n| n < 3 }   # [3, 4, 5]

# find returns first matching value
numbers.find { |n| n > 3 }     # 4

Hashes

  • Key–value data structure
  • Keys are usually symbols or strings
# Create a hash
user = {
  "name" => "Alice",
  "age"  => 25
}

# Access value
user["name"]   # "Alice"

# Add a key-value pair
user["email"] = "alice@example.com"

# Remove a key-value pair
user.delete("age")

Symbols (:symbol)

  • Lightweight names used as labels (not variables)
  • Common as hash keys and method arguments
# Symbol used as a hash key
user = {
  name: "Alice",
  age: 25
}
user[:name]   # "Alice"

Conditionals

if, elsif, else

  • Modifier form for simple one-liners
  • Standard (e.g. if … end) for multi-line logic
  • if uses Truthiness (only false and nil are false). See Page 2
# Standard if statement
if total > 40
    puts "Large order"
end

# Ruby-style modifier (one line)
puts "Large order" if total > 40

# Using elsif and else
if total > 100
  puts "Free shipping"
elsif total > 40
  puts "Standard shipping"
else
  puts "Shipping applies"
end

Case / When (conditional branching)

  • Compares one value against many options
  • Cleaner than long if / elsif chains
  • Uses when instead of elsif
status = :paid    # value being checked
case status
when :paid
  puts "Order paid"
when :pending
  puts "Awaiting payment"
else
  puts "Unknown status"
end

Comparison & Logical Operators

  • Used to compare values and in loops
  • Return true or false
count = 10
limit = 5

count == limit     # equal? false
count != limit     # not equal? true
count > limit      # greater than? true
count <= limit     # less or equal? false

count > 5 && limit < 10   # both true
count < 5 || limit < 10   # either true
!false            # negation → true

Loops/Iteration (most used only)

# Each: Iterates over an array:
orders.each do |order|
    puts order[:id]
end

# Times: Repeats a block X times
5.times do |i|
    puts "Iteration #{i + 1}"
end

Methods

  • Reusable blocks of code that perform a task
  • Can take arguments (parameters)
  • Last expression is returned automatically
  • return is optional and exits immediately
  • Parentheses optional with no arguments
def total(price, qty)
    price * qty
end
